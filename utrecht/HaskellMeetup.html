Light Verification

Liquid Haskell is a refinement type checker for Haskell! You can run it

  • in the browser (e.g., using this link),
  • in the command line (after installation from Hackage) or github,
  • or as a GHC plugin (-fplugin=LiquidHaskell).

You can use it to automatically prove light program properties (e.g., no division by zero) or to manually prove deep program properties (e.g., correctness of mapRecude).

Let’s how it works!

This is a Haskell module that you can run in your browser:

module HaskellMeetup where {-@ LIQUID "--reflection" @-} {-@ LIQUID "--ple" @-} import Language.Haskell.Liquid.ProofCombinators import Prelude hiding (take, drop)

Refinement Types

Liquid Haskell comes with refinement types for various built in Haskell functions. For example, division by zero is not allowed:

div :: Integral a => x:a -> {v:a | v != 0} -> a

Refinement Types, like {v:a | v != 0}, are here, Haskell types refined with logical predicates.

Each call to div will check that the second argument is not zero.

test0 :: Int -> Int test0 x = 42 `div` 2

Note, the check is automatically performed by an external SMT solver (Z3 by default).

Pre- and Post-Conditions

{-@ take :: i:Nat -> xs:{[a] | i < len xs}
         -> {v:[a] | len v = i} @-}
{-@ test1 :: [{v:Int | v /= 0 }] -> Int -> Maybe [Int] @-} test1 :: [Int] -> Int -> Maybe [Int] test1 xs i = if 0 <= i && i < length xs then Just (map (42 `div`) (take i xs)) else Nothing

Verification with Liquid Haskell is

  • path sensitive,
  • type based,
  • SMT automated!

Recursive Functions

Let’s now define the take function.

{-@ take :: i:Nat -> xs:{[a] | i < len xs} -> {v:[a] | len v = i} @-} take :: Int -> [a] -> [a] take 0 _ = [] take _ [] = [] take n (x:xs) = x : take (n-1) xs

When defining a function, Liquid Haskell requires both the refined and unrefined types. (GHC does not see the refinement types, which are provided as comments.) But, for each function Liquid Haskell checks that is terminating and total (which can be deactivated with the --no-termination and --no-totality flags).

Dually, let’s define the drop function.

{-@ drop :: i:Nat -> xs:{[a] | i < len xs} -> {v:[a] | len v = len xs - i} @-} drop :: Int -> [a] -> [a] drop 0 xs = xs drop _ [] = [] drop n (_:xs) = drop (n-1) xs

Can we now prove that take and drop reconstruct the original list? Yes, using deep verification!
